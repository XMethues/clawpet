# clawchat-pet

clawchat-pet 把 Hermes 中的 Agent 活动解释为宠物持续发展的共享成长旅程，并通过当前玩法场景把宠物的活动呈现给主人。宠物主要来自 Petdex，`yinyue-2` 是默认宠物而不是特殊领域角色。

## Language

**Hermes 活动**:
Agent 在 Hermes 中发生的可观察行为或生命周期变化，是成长事件和活动状态的共同来源。
_Avoid_: 成长事件、活动状态

**宠物**:
承载共享成长进度和活动呈现的当前角色；宠物主要来自 Petdex，未选择时使用 `yinyue-2`，切换宠物不会重置进度或玩法场景。
_Avoid_: 银月

**Petdex**:
clawchat-pet 的主要宠物来源。

**共享成长进度**:
独立于当前宠物和玩法场景的持久成长旅程，由所有宠物和玩法场景共享；它只保存稳定成长事实，不保存任何玩法场景的名称或成品文本。
_Avoid_: 修炼进度、宠物存档、场景存档

**稳定成长事实**:
完成去重和结算后形成的场景中立记录，只包含语义标识、数值和关系；名称、话语、提示与纪事文本由玩法场景生成，不属于稳定成长事实。
_Avoid_: 修仙纪事、展示文本

**成长阶段**:
共享成长旅程中的有序位置，与玩法场景无关；玩法场景可以为同一成长阶段使用不同名称。
_Avoid_: 境界、航段

**成长维度**:
构成共享成长进度的场景中立数值含义；玩法场景只为成长维度提供主题名称，不改变其数值或结算规则。
_Avoid_: 灵气、心魔、航程数据

**成长快照**:
共享成长进度在当前时刻的场景中立摘要；玩法场景结合成长快照与近期稳定成长事实生成宠物呈现。
_Avoid_: 场景快照、事件重放结果

**玩法场景**:
对稳定成长事实和活动状态的一套主题解释；一个玩法场景拥有多个可用皮肤和一个默认皮肤。切换玩法场景只改变名称、话语和呈现，并把当前皮肤设为该场景的默认皮肤，不改变结算规则或进度。
_Avoid_: 皮肤、规则体系

**修仙场景**:
默认玩法场景，把共享成长解释为境界、灵气、心魔、历练和道场纪事。
_Avoid_: 核心规则

**皮肤（skin）**:
隶属于一个玩法场景的视觉方案，定义界面的颜色、背景和视觉氛围；主人可以覆盖它的视觉值。皮肤不命名成长事实，也不改变成长规则。
_Avoid_: 玩法场景、规则效果

**宠物呈现**:
把当前宠物、活动状态和共享成长进度按当前玩法场景及皮肤组合成的单一读取结果；界面只显示宠物呈现，不自行拼接多个领域接口。
_Avoid_: 状态拼盘、多接口快照

**宠物话语**:
当前宠物以文字气泡呈现的话语，使用 Petdex 的 `displayName` 作为说话者；它不是音频。
_Avoid_: 宠物语音

**宠物人格**:
按 Petdex 宠物分别保存、由主人修改或生成的话语风格；它只影响宠物话语，不改变成长规则、进度或玩法场景。切换到尚未决定人格的宠物时，应询问主人是否创建；若主人跳过，该宠物使用中立话语且不重复询问。

**成长事件**:
由一次 Hermes 活动解释而来的规范化、场景中立领域事实；它不包含 Hermes 原始载荷，只结算一次，并可能改变共享成长进度或产生稳定成长事实。
_Avoid_: 活动状态

**活动状态**:
当前宠物所有在途活动的综合短时呈现，而非最后一条事件的标签；它可以随时间自然老化，但老化本身不会产生成长事件。
_Avoid_: 成长事件

**候旨**:
修仙场景对“等待主人作出人工批准决定”这一活动状态的称呼；smart-mode 自动判断不属于候旨。
_Avoid_: 自动批准判断

**推演**:
修仙场景对“Agent 正在思考或进行 smart-mode 自动判断”这一活动状态的称呼。
_Avoid_: 候旨

**分神化身**:
修仙场景对“子 Agent 正在代为执行任务”这一活动状态的称呼；它本身不产生额外成长收益，收益由父级委派活动结算。
_Avoid_: 独立成长事件

**历练失败**:
修仙场景对“工具实际执行后发生错误所形成的成长事件”的称呼；主人拒绝批准或批准超时不属于历练失败。
_Avoid_: 批准拒绝、批准超时

**历练结果未明**:
修仙场景对“工具已经结束但现有结果不足以可靠判断成功或失败”的称呼；它不产生奖励或惩罚。
_Avoid_: 历练成功、历练失败
