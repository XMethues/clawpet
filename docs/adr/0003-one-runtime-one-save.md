# 一个运行时拥有一个产品存档

clawchat-pet 由一个进程内运行时拥有 `save.json`、唯一锁和原子提交。共享成长 module 唯一负责创建、加载、验证、结算和序列化共享成长状态，根运行时把该状态视为不透明值，只负责将其与其他产品状态一同原子保存；持久写入失败时直接丢弃本次事务对象。Hermes Hook adapter 把原始回调交给 Hermes 活动 module 完成关联与判断，只有其产出的规范化成长事件可以进入共享成长 module；共享成长不接受 Hermes 原始载荷。Hermes 活动 module 只负责内存中的短时关联，运行时只在持久提交成功后确认对应的短时变化。所有共享成长读取和修改都由共享成长 module 自动先结算到当前时刻，调用者不能单独驱动时间衰减；若读取产生持久变化，运行时在返回前原子保存。共享成长、当前宠物、宠物人格以及场景/皮肤选择都通过这个事务边界修改；Petdex 文件只作为可重建缓存。Hermes Hook 直接调用该运行时，HTTP 仅适配 `/presentation`、`/catalog`、`/command`、宠物图片和健康检查。短时活动重启后回到 idle。新实现只验证并读取当前存档格式，不识别旧格式类别，也不包含自动重置、迁移或转换分支；非当前格式文件必须在部署时手动删除。旧 HTTP 接口同样不兼容。
